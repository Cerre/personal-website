Personal Website with Agentic LLM Tools

Python HTML/CSS LLM AI Agents
GitHub Repository Live Demo

Problem Statement

As a technical professional, I needed a portfolio website that effectively showcased my skills and projects. However, I wanted to use this opportunity to explore the emerging capabilities of agentic LLM tools in software development—venturing beyond my established knowledge domain of Python and machine learning.

The challenge was to create a professional portfolio website while experimenting with and evaluating how effectively AI agents could assist in web development tasks that were outside my core expertise.

Technical Approach

1. Defining the Agency Framework

I established a collaborative human-AI workflow for the project:

  • Set clear boundaries between human-directed creative decisions and AI-assisted implementation
  • Defined project requirements, visual aesthetics, content strategy, and success metrics
  • Created a structured conversational framework for interacting with the AI agent
  • Developed Python scripts to facilitate and process LLM API interactions
  • Established version control and deployment processes for iterative development

2. Website Structure and Content Strategy

Working with the AI agent, I developed:

  • A responsive single-page design with expandable project case studies
  • A coherent information architecture focusing on professional expertise
  • A technical case study format highlighting challenges, approaches, and insights
  • A content strategy that emphasized technical depth while maintaining accessibility

3. Implementation Process

The development followed an iterative process:

  • Initial scaffolding of HTML/CSS structure with AI assistance
  • Python-based automation for interacting with LLM APIs and processing responses
  • Progressive refinement through conversational guidance
  • Component-by-component development with human review at each stage
  • Human-directed modifications based on visual inspection and usability considerations
  • Deployment to GitHub Pages with automated build process

4. Technical Case Study Development

For the project case studies, I implemented:

  • Detailed technical narratives structured around problem-solving processes
  • In-depth explanations of technical approaches and methodologies
  • Visual hierarchy emphasizing key technical insights
  • Navigation systems allowing exploration of multiple projects

Technical Challenges

Challenge 1: Balancing AI Assistance with Human Direction

Finding the right level of guidance to provide the AI was initially difficult—too specific and it limited creativity; too vague and the results required extensive revision.

Solution: I developed a tiered approach to collaboration where I would first establish clear objectives and constraints, then ask the AI to propose implementation strategies, evaluate these proposals, and finally guide the detailed implementation. This created a consistent feedback loop that maintained progress while ensuring quality.

Challenge 2: Technical Knowledge Transfer

Effectively articulating web development requirements as someone without extensive HTML/CSS experience was challenging.

Solution: I used a progressive learning approach, where both the AI and I built shared terminology and understanding through iterative development. Starting with basic components and gradually increasing complexity allowed me to learn sufficient terminology to direct later, more complex development tasks.

Challenge 3: Maintaining Coherence Across Iterations

As development progressed through multiple sessions, maintaining consistency in design and functionality became difficult.

Solution: I implemented a documentation strategy where each major change was clearly noted with its purpose and implementation details. This created a shared "memory" that could be referenced in future sessions, ensuring design decisions remained consistent even across multiple development interactions.

Results and Insights

The project produced valuable outcomes beyond just the website itself:

  • A professional portfolio website completed with minimal prior web development experience
  • Development time reduced by approximately 60% compared to traditional learning and implementation
  • A modular, maintainable codebase that can be easily extended for future needs
  • A documented methodology for using AI agents for tasks outside one's expertise domain

Key technical insights gained:

  1. Effective prompt engineering is crucial when working with AI agents—well-structured requests with clear constraints produce significantly better results
  2. Domain-specific terminology acquisition is a necessary first step in working with AI in unfamiliar areas
  3. Building knowledge incrementally leads to better results than attempting to complete complex tasks immediately
  4. The most effective workflow involves human creative direction combined with AI implementation assistance, rather than fully delegating creative decisions

Future Improvements

This project established a foundation for continued exploration:

  • Interactive Elements: Adding JavaScript functionality for more dynamic user experiences
  • Systematic Design System: Developing a more comprehensive design system for consistent visual language
  • Automated Content Updates: Creating a streamlined process for adding new projects and updates
  • Performance Optimization: Further refining loading times and responsiveness
  • Accessibility Enhancements: Implementing additional features to improve site accessibility

Most importantly, this project demonstrated that with appropriate human-AI collaboration frameworks, it's possible to effectively venture beyond established knowledge domains and produce high-quality results in new technical areas.
